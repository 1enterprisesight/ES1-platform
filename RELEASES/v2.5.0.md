# ES1 Platform v2.5.0 — Release Notes

**Date:** 2026-03-12
**Type:** Rebaseline release — clean verified baseline for all deployments

## Images Changed (vs v2.4.0)
- sentinel-api: **NEW** — AI data dashboard backend
- sentinel-ui: **NEW** — AI data dashboard frontend
- platform-manager-api: Redis resilience, knowledge sync, SSE fix
- platform-manager-ui: (rebuilt with latest)
- airflow: Upgraded base to Apache Airflow 3.1.7 (from 2.8.1)
- All images rebuilt clean as amd64 for production baseline

## Database Migrations Included
### Main PostgreSQL (4 migrations: V001–V004)
- V001: Extensions (uuid-ossp, pgcrypto, vector) + app schema
- V002: Audit schema (api_requests, security_events, api_keys) — **REQUIRED by Platform Manager**
- V003: Audit grants fix
- V004: Change sets for gateway config management

### AIML PostgreSQL (12 migrations: V001–V012)
- V001–V002: Extensions (pgvector, pg_trgm) + schemas (vectors, rag, agents, mlops)
- V003–V004: Vector tables (1536/384/768/4096 dim embeddings) + RAG tables
- V005–V006: Agent definitions/sessions/memory + MLops model registry
- V007: Knowledge graph schema (entities, relationships, traversal functions)
- V008: Knowledge base columns fix
- V009–V012: **Sentinel schema** (users, sessions, datasets, workspaces, workspace-scoped refactor)

## Seed Data (loaded by db-migrate --seed=reference)
- 9 entity types for knowledge graph (person, org, location, concept, etc.)
- 10 relationship types (works_for, located_in, part_of, etc.)
- Default "documents" knowledge base + local-uploads source
- 21 template agents (CrewAI teams: research, code review, support; AutoGen teams: code assistant, debate, brainstorm)

## Configuration Changes
- KrakenD: Sentinel routes added (/api/v1/sentinel/*)
- Helm: sentinel.enabled flag added to infrastructure chart
- Helm: All per-service image tags cleared — global.imageTag now controls all images
- Airflow: API endpoint changed /api/v1 → /api/v2, component renamed webserver → api-server
- Release workflow: now data-driven from release-config.yaml (no hardcoded image lists)
- Upstream images: all pinned to specific versions (no more :latest tags)

## Breaking Changes
- Airflow 2.x → 3.x: Schedule parameter, auth model, API version all changed
- Image tags: All services now use global.imageTag (v1.7.0 hardcoded tags removed)

## Cross-Service Dependencies
- Platform Manager → audit.api_keys (postgres V002)
- Sentinel → sentinel.* schema (aiml-postgres V009-V012)
- Airflow DAG bundle → platform_manager.dag_files
- Sentinel requires GEMINI_API_KEY in secrets

## Service Startup Order
1. PostgreSQL + AIML PostgreSQL (StatefulSets)
2. Redis
3. db-migrate (pre-install Job — migrations + seeds on BOTH databases)
4. Platform Manager API (requires audit schema, seeds DEFAULT_API_KEY)
5. Airflow (requires PostgreSQL for metadata + platform_manager for DAG bundle)
6. KrakenD (routes to all backends)
7. Sentinel (requires sentinel.* schema, bootstraps admin user)
8. All other services (Langflow, Langfuse, n8n, Ollama, MLflow, monitoring)

## Upgrade Steps (from v2.4.0 or earlier)
1. Update deployment.yaml version to v2.5.0
2. Load images: `./es1 images load <deployment-id>`
3. Upgrade: `./es1 upgrade <deployment-id>` (runs migrations automatically via Helm hook)
4. Verify: `./es1 verify <deployment-id>`

## Fresh Install Steps
1. Provision cluster: `./es1 provision <deployment-id>`
2. Generate secrets: `./es1 secrets generate <deployment-id>`
3. Load images: `./es1 images load <deployment-id>`
4. Install: `./es1 install <deployment-id>`
5. Setup: `./es1 setup <deployment-id>`
6. Verify: `./es1 verify <deployment-id>`
